# Лабораторная работа 4.Бузмакова Кира ИТ-16. Вариант 5

## Задача 1: Коробка (Box)

**Условие задачи:**  
Создать сущность Коробка, которая может хранить один произвольный объект в один момент времени. Объект можно получить и разместить на хранение в любой момент времени. Если объект забирают из коробки – ссылку на этот объект необходимо обнулить. Если объект кладут в коробку, но она не пуста – необходимо выкинуть исключение. Имеет метод проверки на заполненность. Методы класса должны работать с тем типом данных, который был указан во время создания объекта.

**Алгоритм решения:**  
Создается обобщенный класс Box с приватным полем для хранения объекта. Конструктор инициализирует коробку как пустую. Метод put проверяет, что передаваемый объект не null и что коробка пуста, затем сохраняет объект. Метод get возвращает объект и обнуляет ссылку. Метод isFull проверяет наличие объекта в коробке.

## Задача 2: Хранилище (Storage)

**Условие задачи:**  
Создать сущность Хранилище, которая может хранить один произвольный объект в один момент времени. Хранилище неизменяемо. Объект кладется в Хранилище при его создании. В качестве объекта может быть сохранено также и значение null. Хранилище может вернуть ссылку на Объект. Если вместо объекта хранится null, необходимо вернуть какое-либо альтернативное значение. Метод получения значения должен работать с тем типом данных, который был указан во время создания объекта.

**Задания для демонстрации:**  
- Хранилище чисел с null (альтернатива 0)
- Хранилище чисел с 99 (альтернатива -1)
- Хранилище строк с null (альтернатива "default")
- Хранилище строк с "hello" (альтернатива "hello world")

**Алгоритм решения:**  
Создается неизменяемый класс Storage с final полем для хранения объекта. Конструктор принимает объект (может быть null). Метод get возвращает сохраненный объект. Метод getOrDefault возвращает альтернативное значение если сохранен null.

## Задача 3: Коробка с трехмерной точкой

**Условие задачи:**  
Создать метод, принимающий Коробку из задачи 1, и кладет в неё трехмерную точку координат с произвольными значениями. Метод должен позволять передавать Коробку с более чем одним видом параметризации.

**Алгоритм решения:**  
Создается класс Point3D для представления трехмерной точки. Реализуется обобщенный метод putPointInBox, принимающий Box<T>. Используется приведение типов для работы с разными видами параметризации. Обрабатываются возможные исключения.

## Задача 4: Функция (FunctionApplier)

**Условие задачи:**  
Разработать метод, который будет принимать список значений типа T, и объект имеющий единственный метод apply. Данный метод надо применить к каждому элементу списка, и вернуть новый список значений типа P, при этом типы T и P могут совпадать, а могут не совпадать.

**Задания для демонстрации:**  
- Передать список строк и получить список чисел (длины строк)
- Передать список чисел и получить список абсолютных значений
- Передать список массивов целых чисел и получить список максимальных значений

**Алгоритм решения:**  
Создается класс FunctionApplier с обобщенным методом applyFunction. Используются два параметра типа: <T, P>. Функция применяется к каждому элементу через Function<T, P>. Возвращается новый список с преобразованными значениями.

## Задача 5: Фильтр (Filter)

**Условие задачи:**  
Разработать метод, который будет принимать список значений типа T и объект имеющий единственный метод test (принимает T и возвращает boolean). Вернуть новый список типа T, из которого удалены все значения не прошедшие проверку условием.

**Задания для демонстрации:**  
- Отфильтровать строки длиной менее 3 символов
- Отфильтровать положительные элементы из списка чисел
- Отфильтровать массивы, в которых нет положительных элементов

**Алгоритм решения:**  
Создается класс Filter с методом filterList. Используется Predicate<T> для проверки условий. Проходится по списку и добавляются только элементы, прошедшие проверку. Возвращается новый список того же типа.

## Задача 6: Сокращение (Reducer)

**Условие задачи:**  
Разработать метод, который будет принимать список значений типа T и способ с помощью которого список значений можно свести к одному значению типа T, которое и возвращается из метода. Метод должен гарантированно не возвращать null и не выбрасывать ошибок в случае пустого списка.

**Задания для демонстрации:**  
- Объединить строки в одну большую строку
- Вернуть сумму всех чисел списка
- Получить общее количество элементов во всех списках
- Проверить работу с пустым списком

**Алгоритм решения:**  
Создается класс Reducer с методом reduce. Используется BinaryOperator<T> для операции сокращения. Добавляется параметр identity для пустых списков. Накопительно применяется оператор к элементам списка.

## Задача 7: Коллекционирование (Collector)

**Условие задачи:**  
Разработать метод, который будет возвращать коллекцию типа P со значениями типа T. Метод будет принимать список исходных значений, способ создания результирующей коллекции и способ передачи значений исходного списка в результирующую коллекцию.

**Задания для демонстрации:**  
- Разделить числа на положительные и отрицательные списки
- Сгруппировать строки по длине
- Получить набор уникальных строк (удаление дубликатов)

**Алгоритм решения:**  
Создается класс Collector с методом collect. Используется Supplier<P> для создания коллекций. Применяется Function<T, Boolean> для условий добавления. Поддерживаются разные типы коллекций через обобщения.
